What does this test do?

This test attempts to trigger parsing errors in middle boxes that parse the HTTP protocol. This indicates that the network under analysis has deep packet inspection (DPI) capabilities. It can also often lead to fingerprinting the version and model of interference equipment used by a certain network. For example, the use of BlueCoat has been detected through this test. You can learn about BlueCoat here (link to report). This test sends out invalid HTTP requests and verifies if a control back-end is able to receive them and if it's not able to, then there's a sign of tampering.
